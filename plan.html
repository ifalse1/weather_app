Weather

Phase 0: Requirements Specification

The focus behind this app is to create a website that will fetch the users ipgeolocation and then using that display the current weather and 5 day forecast for the area. The user can then vote on if they think a condition is unlikely, nuetral, or likely for the forecast.

This assignment is to be written using the vue.js framework.

If there are any errors with fetching data those need to be displayed to the user. A loading message also needs to be displayed.

Phase 1: Design

The first step is to make the appropriate api calls and use a promise chain to recieve the data. While it is fetching the data a loading message will be displayed to the user. If any errors happen then display an appropriate error message

Once the data is recieved first display the location of the user. The minimum data to be displayed is the latitude or longitude but also display the city, state, and country if applicable

Next display the current weather conditions. The data needed to be displayed:

  • Current Temperature
  • High Temperature
  • Low Temperature
  • Sky Conditions (i.e. clear, rainy, etc.)
  • The humidity
  • The pressure

The final thing to display is the 5 day 3 hour forecast. Each of these display the same information as the current weather condition The user can vote on wether these are likely, nuetral, or unlikely with each one of these changing the class using vue class binding.
